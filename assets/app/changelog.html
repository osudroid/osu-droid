# Additions - Add beatmap attribute display by [Rian8337](https://github.com/Rian8337) - Can be displayed by pressing the text with difficulty values (CS, AR, OD, HP, star rating) in song select. - Add option for selecting language in settings by [Rian8337](https://github.com/Rian8337) - Can be accessed under the `General` category in settings. - Changing the language requires restarting the game. Therefore, it is disabled in multiplayer. - Add ability to search mods and mod presets in mod menu by [Rian8337](https://github.com/Rian8337) # Changes - Do not update OD value when using Precise mod in song select by [Rian8337](https://github.com/Rian8337) - It's the hit window that is different, not the OD, so attempting to "pair" OD like this is conceptually wrong. - Rework input handling in gameplay by [Rian8337](https://github.com/Rian8337) - This solves an issue where inputs between two frames (commonly known as subframe inputs) are discarded except the last input before the next frame (an input is either a finger press, finger drag, or finger lift). - Makes hitobject judgements "consume" finger presses sequentially. This means that if there are `n` hitobjects with the same starting time, and they overlap, they would require `n` presses to hit instead of one (there is a [beatmap](https://osu.ppy.sh/beatmapsets/1157609#osu/2439789) that abuses this behavior). - Numerous optimizations to difficulty calculation by [Rian8337](https://github.com/Rian8337) - May not be noticeable, but should be better regardless. Maybe the improvement is noticeable in gameplay loading if real-time pp counter is enabled. - Localization updates by community contributors ## osu!droid Star Rating and Performance Points Changes A newspost containing more details about these changes will be published after they are applied to global ranking. - Replace existing visual skill with a better implemented reading skill by [Rian8337](https://github.com/Rian8337) - This is a port of [this](https://github.com/ppy/osu/pull/33196) change with but modified to fit osu!droid's gameplay environment and existing system. - Notable adjustments: - Reduced high AR difficulty since approach circle starts at 3x scale instead of 4x - Partial slider reading difficulty assessment from visual skill is kept with some changes to constants for balancing - Rhythm is accounted for via rhythm difficulty calculation - Miss penalty additionally accounts for a proportion-based miss penalty to account for cheesing - Accuracy scaling uses estimated unstable rate (lower OD gives a harsher scaling) instead of accuracy percentage - Adjust angle bonuses in aim to balance very difficult aim scores gaining a lot of pp from reading skill by [Rian8337](https://github.com/Rian8337) - Cap vibro tapping difficulty delta time at 300 1/4 BPM interval by [Rian8337](https://github.com/Rian8337) - Scores with note intervals higher than said interval will receive more pp, provided that they are above a certain estimated unstable rate. # Bug Fixes - Fix potential crash when canceling gameplay loading while slider paths are being loaded by [Rian8337](https://github.com/Rian8337) - Fix non-host players being able to change required mods' settings in free mod by [Rian8337](https://github.com/Rian8337) - Fix score overflow when playing a beatmap with information that has not been fully populated by [Rian8337](https://github.com/Rian8337) - Full population happens in background difficulty calculation. This bug happens when a player plays a beatmap with the ScoreV2 mod enabled and with the beatmap not being calculated yet. - Fix replay import notifications not showing by [Rian8337](https://github.com/Rian8337) - Fix Really Easy mod approach rate adjustment only counting Custom Speed mod instead of all constant rate-changing mods by [Rian8337](https://github.com/Rian8337) - Fix wrong displayed score when opening it from pp-based beatmap leaderboard if the score is different from the score-based leaderboard by [Rian8337](https://github.com/Rian8337) - Fix room host being able to clear ScoreV2 mod in multiplayer by [Rian8337](https://github.com/Rian8337) - Fix incorrect overall difficulty in Relax osu!droid performance calculation by [Rian8337](https://github.com/Rian8337) - Fix show scoreboard setting potentially only taking effect after restarting gameplay by [Rian8337](https://github.com/Rian8337) - Fix beatmap reimport (e.g., redownloading a beatmap in beatmap downloader) not working due to import redundancy checks by [Rian8337](https://github.com/Rian8337) # Removals - Remove `Remove slider lock and spinner lock` setting by [Rian8337](https://github.com/Rian8337) - With the rework to gameplay input handling, this is enabled by default and thus is no longer needed.
